使用hexo制作个人博客，并使用GitHub Pages托管中遇到的一些问题和解决方法

2025/9/10

前言

第一次尝试使用Hexo搭建自己的静态博客，选择了ParticleX主题，链接为https://github.com/theme-particlex/hexo-theme-particlex，同时将博客托管在Github Pages上，过程并不困难，但是还是分享一下笔者遇到的一些问题以及解决方法，而并不是给出搭建步骤。

1. Github创立仓库时的命名问题

1.1 Github Pages仓库类型

Github Pages支持两种类型的仓库，一种是用户/组织站点（User/Org site），一种是项目站点（Project site）。
用户/组织站点（User/Org site）命名为{username}.github.io，注意，你的站点是哪种站点是根据你的命名来的，你命名成这种格式就是用户/组织站点。，这种站点的最后url一般是https://username.github.io，
项目站点（Project site）的仓库名可以是除了上述情况下的任意名字，站点的最后url一般是https://username.github.io/{repository}/。

1.2 对 Hexo 博客的影响

1.2.1 情况1：用户站点

假设仓库名是{username}.github.io，那么在项目中的配置文件_config.yml应该设置
url: https://{username}.github.io
root: / (有时候是path,具体看文件中是如何命名)
这样资源路径可以直接用 /css/main.css，首页 URL 就是 https://waterwo0d.github.io，Hexo 生成文件直接部署到 master 或 main 分支即可

1.2.2 情况2：项目站点

只要你的仓库名不是{username}.github.io，就算是username.github.io(注意，这里没有用{}进行转义，和{username}.github.io是不一样的)。举个例子，假如你的用户名是abc,那么{username}.github.io就是abc.github.io,而不是name.github.io。
那么在项目的配置文件_config.yml文件中，应该设置
url: https://{username}.github.io/{repository}
root: /{repository}/ (有时候是path,具体看文件中是如何命名)
所有静态资源引用都要加上 root 前缀，不然资源无法加载。

1.2.3 实际情况

Hexo 中 root 默认是 /，表示相对于站点根目录，而很多第三方主题都会在资源引用中使用<%= url_for(…) %>，这个会自动加上仓库路径，这种情况下，就算你的站点是项目站点，也不需要设置root: /{repository}/ ，如果加了，反而会导致路径加载错误。例如，假设仓库名称是waterwood.github.io，这是项目站点，使用了ParticleX主题，如果再添加root: /waterwood.github.io/，那么就会导致资源路径重复，变成 /waterwood.github.io/waterwood.github.io/css/…，所以访问会出错。
所以在添加root或者path时，一定要清楚资源的加载路径是什么，如果不想读代码，也可以两个都试试，看看哪个成功加载（汗）。
